# المنظومة البيقونية — منصة متكاملة (Backend حقيقي)

منصة ويب فيها:
- 🏠 **صفحة رئيسية (Hub)** بقائمة جانبية تجمع الأقسام الستة (العرض التفاعلي، الكتاب التفاعلي، التلخيصات، الخريطة الذهنية، اختبار الحفظ، اختبارات الشرح).
- 🔐 **تسجيل دخول حقيقي**: تسجيل حساب جديد + دخول، بكلمات مرور مُشفّرة (bcrypt) وجلسات محفوظة في قاعدة البيانات.
- 🛠️ **لوحة تحكم للمطور (Admin)**: رفع ملفات جديدة، تعديل محتوى أي صفحة مباشرة، استبدال الملف، حذف الصفحة، ومتابعة قائمة المستخدمين.
- 🗄️ **قاعدة بيانات حقيقية** (SQLite عبر better-sqlite3) — كل شيء بيتخزن في ملف `db/app.db`، مش في المتصفح.

---

## 1) المتطلبات

- Node.js نسخة 18 أو أحدث (مجرّب على v22).
- npm (بييجي مع Node تلقائيًا).

## 2) التركيب والتشغيل (على جهازك)

```bash
# داخل مجلد المشروع
npm install          # تثبيت المكتبات
npm run seed         # تجهيز قاعدة البيانات وإدخال الأقسام الستة
npm start            # تشغيل السيرفر
```

بعد التشغيل هتلاقي رسالة:
```
✅ المنظومة البيقونية شغالة على http://localhost:3000
```

افتح المتصفح على `http://localhost:3000` — هيوجّهك تلقائيًا لصفحة تسجيل الدخول.

### أول حساب = مطوّر تلقائيًا

**أول حساب** يتسجل من صفحة "إنشاء حساب" بيتحول تلقائيًا لصلاحية **مطوّر (admin)** وبيشوف رابط "لوحة تحكم المطور" في القائمة الجانبية. أي حساب بعده بيبقى "مستخدم عادي" (يقدر يتصفح المحتوى بس، من غير رفع أو تعديل).

> ⚠️ لو عايز تضيف مطوّرين تانيين بعد كده، ده محتاج تعديل بسيط في قاعدة البيانات (عمود `role` في جدول `users`) — ينفع نضيفلك صفحة "ترقية مستخدم" من داخل لوحة التحكم لو حبيت.

## 3) هيكل المشروع

```
beeqouniyah-app/
├── server.js              # نقطة تشغيل السيرفر (Express)
├── package.json
├── .env.example           # انسخه لـ .env وغيّر القيم
├── db/
│   ├── init.js             # الاتصال بقاعدة البيانات + إنشاء الجداول
│   ├── seed.js             # تعبئة الأقسام الستة أول مرة
│   └── app.db               # ملف قاعدة البيانات (بيتعمل تلقائيًا)
├── routes/
│   ├── auth.js              # تسجيل / دخول / خروج
│   ├── content.js           # عرض قائمة الصفحات ومحتواها (محمي بتسجيل الدخول)
│   └── admin.js             # رفع / تعديل / حذف الصفحات (محمي بصلاحية المطوّر)
├── middleware/
│   └── auth.js              # requireAuth / requireAdmin
├── content/                 # ملفات الأقسام الفعلية (html) — هنا بيتم التخزين والتعديل
└── public/                  # واجهة الموقع
    ├── index.html            # الصفحة الرئيسية (Hub)
    ├── login.html
    ├── register.html
    ├── admin.html            # لوحة تحكم المطوّر
    ├── css/style.css
    └── js/{app.js, admin.js}
```

## 4) إزاي شغالة تقنيًا

- **الجلسات (Sessions)**: بتتخزن في نفس قاعدة بيانات SQLite (جدول `sessions`)، مش في localStorage، فبتفضل شغالة حتى لو قفلت المتصفح، وآمنة أكتر من التخزين المحلي.
- **كلمات المرور**: بتتشفّر بـ bcrypt قبل ما تتخزن، مفيش كلمة مرور بترجع نص صريح في أي وقت.
- **حماية المحتوى**: أي صفحة من الأقسام الستة (وأي صفحة تضيفها) بتتقرأ عن طريق مسار API محمي (`/api/content/pages/:slug`)، مش ملف static عام — يعني محدش يقدر يوصلها من غير تسجيل دخول.
- **صلاحيات المطوّر**: أي مسار تحت `/api/admin/*` بيتفحص إن `role === 'admin'` قبل ما ينفذ أي حاجة.
- **رفع الملفات**: عن طريق multer، بيتقبل بس الأنواع دي: `html, htm, txt, pdf, png, jpg, jpeg, svg`، وبيتفحص اسم الملف عشان محدش يرفع حاجة برة مجلد `content/` (حماية من Path Traversal).

## 5) نشر الموقع على الإنترنت (Deployment)

المشروع ده سيرفر Node.js حقيقي، يعني محتاج استضافة بتشغّل Node (مش استضافة استاتيك عادية). أسهل الخيارات:

- **Railway** أو **Render**: اربط المشروع بـ GitHub، وحدد أمر التشغيل `npm start` وأمر البناء `npm install && npm run seed`.
- **VPS عادي** (DigitalOcean / Hetzner / أي سيرفر لينكس): ارفع المشروع، شغّل `npm install && npm run seed`، وبعدين استخدم `pm2` عشان السيرفر يفضل شغال:
  ```bash
  npm install -g pm2
  pm2 start server.js --name beeqouniyah
  pm2 save
  ```
  وحط Nginx قدامه كـ reverse proxy لو عايز دومين وHTTPS.

**قبل النشر لازم:**
1. تنسخ `.env.example` لـ `.env` وتحط `SESSION_SECRET` قيمة عشوائية طويلة وسرية.
2. تتأكد إن `db/app.db` ومجلد `content/` بيتعملهم نسخة احتياطية (backup) بشكل دوري، لأنهم قاعدة البيانات والملفات الحقيقية بتاعتك.

## 6) استخدام لوحة تحكم المطوّر

بعد الدخول بحساب المطوّر، هتلاقي زرار "لوحة تحكم المطور" في القائمة الجانبية:

- **رفع صفحة جديدة**: حدد عنوان، أيقونة (إيموجي اختياري)، والملف — وهتتضاف فورًا للقائمة الجانبية لكل المستخدمين.
- **تعديل صفحة**: لو الملف HTML/نص، هتقدر تعدّل الكود مباشرة في مربع نص وتحفظ. لو صورة/PDF، هتقدر بس تستبدلها بملف جديد.
- **حذف صفحة**: بيمسح الملف من السيرفر وبيشيلها من القائمة.
- **قائمة المستخدمين**: عرض فقط، لمتابعة مين اتسجل في المنصة.

## 7) تحديثات جديدة: الصفحة الرئيسية العامة، لوحة الطالب، جدول المحفوظات، وتسجيل الدخول بجوجل

### أ) الصفحة الرئيسية العامة (Landing)
أي زائر (حتى قبل تسجيل الدخول) بيشوف على `/` صفحة احترافية فيها مقال عن أهمية دراسة علم مصطلح الحديث، وزرار "تسجيل الدخول" و"إنشاء حساب". الصفحة متجاوبة بالكامل مع الموبايل.

### ب) لوحة الطالب بعد الدخول
بعد تسجيل الدخول، الصفحة الرئيسية (`/index.html`) بقت شبكة كروت احترافية (بنفس تصميمك اللي بعتهولنا) بدل القائمة الجانبية البسيطة، وفيها كارت خاص لـ **"جدول محفوظاتي"**. القائمة الجانبية لسه موجودة لسهولة التنقل، وكل حاجة متجاوبة مع الموبايل (بتتحول لقائمة منسدلة بزرار ☰).

### ج) جدول المحفوظات
كل طالب عنده صفحة خاصة بيه (`/tracker.html`) فيها الأبيات الأربعة والثلاثين، يقدر:
- يحدد تاريخ مستهدف/إنجاز لكل بيت بنفسه.
- يأشّر البيت كـ"محفوظ" لما يخلص حفظه.
- يشوف نسبة تقدّمه الإجمالية في شريط تقدّم أعلى الصفحة.

البيانات دي خاصة بكل مستخدم على حدة ومحفوظة في قاعدة البيانات (مش بتتشارك بين الطلاب).

### د) تسجيل الدخول بالبريد الإلكتروني
تسجيل الدخول والتسجيل بقيا بالبريد الإلكتروني بدل اسم المستخدم. أول بريد يتسجل بيبقى المطوّر تلقائيًا (زي ما كان قبل كده بالظبط، بس بالإيميل).

### هـ) تسجيل الدخول بحساب Google (اختياري)
تقدر تفعّل تسجيل الدخول بجوجل، بشرط واحد: **جوجل بيسمح بيه بس للحسابات اللي سجّلت بالفعل بالبريد وكلمة المرور** — يعني مش هتقدر حد يعمل حساب جديد بجوجل مباشرة من غير ما يكون مسجل بالإيميل الأول. ده بيحمي حسابات المطوّرين ويمنع تسجيل حسابات عشوائية.

#### إزاي تفعّلها (خطوات إنشاء Google Client ID):

1. روح [console.cloud.google.com](https://console.cloud.google.com) وسجّل دخول بحساب Google عادي.
2. من القائمة فوق، اعمل **مشروع جديد (New Project)** — اديله أي اسم، مثلاً `beeqouniyah`.
3. من القائمة الجانبية: **APIs & Services** → **OAuth consent screen**.
   - اختار **External**، دوس **Create**.
   - املأ اسم التطبيق (مثلاً "المنظومة البيقونية") وإيميلك، واحفظ (Save and Continue) على كل الخطوات من غير ما تضيف حاجة زيادة.
4. من نفس القائمة: **APIs & Services** → **Credentials**.
   - دوس **Create Credentials** → **OAuth client ID**.
   - Application type: اختار **Web application**.
   - في خانة **Authorized JavaScript origins**، ضيف رابط موقعك (المهم يكون بالظبط زي ما هو، من غير `/` في الآخر)، مثلاً:
     ```
     https://byqoniya.bonto.run
     ```
   - دوس **Create**.
5. هيديك **Client ID** (شكله شبه `123456789-abc.apps.googleusercontent.com`) — انسخه.
6. حطه في متغيرات البيئة بتاعت السيرفر (نفس مكان `SESSION_SECRET`):
   ```
   GOOGLE_CLIENT_ID=الرقم-اللي-نسخته
   ```
7. أعد تشغيل السيرفر (Restart). زرار "تسجيل الدخول بجوجل" هيظهر تلقائيًا في صفحة الدخول، وزرار "ربط حساب Google" هيظهر لكل مستخدم في القائمة الجانبية لحد ما يربط حسابه.

> لو سبت `GOOGLE_CLIENT_ID` فاضي، الموقع هيشتغل عادي بس بدون خيار جوجل — مفيش أي مشكلة.

### و) تحديث المشروع اللي منشور بالفعل على Bonto
لو عندك نسخة شغالة على Bonto من قبل، عشان تحدّثها بكل التعديلات دي:
1. افتح المشروع في محرر Bonto.
2. احذف الملفات القديمة اللي اتغيرت (`server.js`, `db/init.js`, `db/seed.js`, `routes/auth.js`, `public/*`) وارفع النسخة الجديدة من الملفات دي (أو استخدم Git push-to-deploy لو ربطته بـ GitHub).
3. من التيرمينال في Bonto، شغّل:
   ```
   npm install
   npm run seed
   ```
4. دوس **Restart** من فوق.

⚠️ **تنبيه مهم**: قاعدة البيانات القديمة كانت بتسجّل المستخدمين بـ"اسم مستخدم" بدل "بريد إلكتروني". الكود الجديد بيحاول ترقية الجدول تلقائيًا، لكن لو كان عندك حسابات تجريبية قديمة، الأسلم إنك تمسح ملف `db/app.db` وتعمل `npm run seed` من جديد وتسجّل حسابك الحقيقي تاني كأول مستخدم (هيبقى مطوّر تلقائيًا زي الأول).
